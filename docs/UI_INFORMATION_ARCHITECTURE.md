# UI Information Architecture

## Goal

Provide a clean, non-overwhelming interface.

---

## Primary Layout

- Header (location + mode toggle)
- Dashboard modules
- Action button
- Navigation tabs

---

## Module Order (LOCKED)

1. Observing Score + Conditions
2. Targets
3. Alerts
4. Passes

---

## UI Modes

- Day
- Night
- Red

---

## Time Context

- Now
- Tonight
- Next 24h

---

## Density Rules

- Max 5 targets
- Max 5 passes
- Max 3 alerts

---

## Interaction Model

- Click → drill down
- No nested complexity
- No cluttered screens
