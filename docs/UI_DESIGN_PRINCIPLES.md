# UI Design Principles

These principles guide all UI work for Astronomy Hub and must be followed across phases.

1. Decision-support first: show what matters now, not everything.
2. Low overload: prefer succinct summaries and drill-downs.
3. Local-first: default to ORAS Observatory and accept only session-only manual overrides in Phase 1.
4. Drill-down over data dumping: provide paths to more detail instead of long lists.
5. Astronomy-safe Red Mode: low-luminance red palette, no bright whites, accessible controls.
6. Mock-first: UI must work with mocked backend responses before introducing live APIs.
7. Mobile-first thinking: dashboard must be useful on small screens.
8. Preserve density rules: capped lists for targets, passes, and alerts.
