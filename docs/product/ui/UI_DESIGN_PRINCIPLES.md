# 🌌 ASTRONOMY HUB — UI DESIGN PRINCIPLES (AUTHORITATIVE)

---

# 0. PURPOSE

This document defines:

> **How the UI must feel, behave, and communicate**

It ensures:

* clarity over complexity
* guidance over confusion
* structure over chaos
* usability over novelty

---

# 1. 🧠 CORE PRINCIPLE

```text id="x4p9k2"
Astronomy Hub is a decision-support system,
not a data display system.
```

---

# 2. 🎯 DESIGN INTENT

The UI must help the user answer:

* What is happening right now?
* What should I look at?
* Why does it matter?
* What should I do next?

---

# 3. 🧭 CLARITY OVER COMPLEXITY

---

## Rule

```text id="c7m2x1"
If something requires explanation, it is too complex.
```

---

## Requirements

* obvious navigation
* clear labeling
* visible actions
* understandable output

---

## Failure if:

* user hesitates
* user must interpret UI behavior
* user must guess meaning

---

# 4. 🎯 SINGLE FOCUS RULE

---

## Rule

```text id="p3k8v6"
The interface must have one primary focus at all times.
```

---

## Implementation

* one active scene
* one active object
* one active context

---

## Failure if:

* multiple competing elements
* user attention split

---

# 5. 📊 HIERARCHY FIRST

---

## Rule

```text id="t5q2x9"
The UI must communicate importance visually, not verbally.
```

---

## Priority Order

1. Scene
2. Selected object
3. Live intelligence
4. Supporting panels
5. Metadata

---

## Failure if:

* everything looks equal
* important elements are hidden

---

# 6. 🧠 GUIDED EXPERIENCE

---

## Rule

```text id="v2m7p4"
The system must guide the user without requiring instruction.
```

---

## Must Provide

* recommendations
* context
* prioritization
* clear next steps

---

## Failure if:

* UI feels like a toolset
* user must decide everything manually

---

# 7. 🔍 CONTEXT ALWAYS VISIBLE

---

## Rule

```text id="k9r4x2"
The user must always know where they are and what they are viewing.
```

---

## Must Show

* current scope
* current engine
* current filter
* selected object

---

## Failure if:

* context is hidden
* user becomes disoriented

---

# 8. 📦 PROGRESSIVE DISCLOSURE

---

## Rule

```text id="q6x3p8"
Show only what is needed now. Reveal more on interaction.
```

---

## Implementation

* simple first view
* deeper layers on click
* detail on demand

---

## Failure if:

* too much information at once
* cluttered interface

---

# 9. 🚫 NO DATA DUMPS

---

## Rule

```text id="m8k2v5"
Data must be structured, prioritized, and contextualized.
```

---

## Must Avoid

* raw data lists
* unstructured text
* overwhelming panels

---

## Failure if:

* UI looks like logs or tables
* user cannot extract meaning

---

# 10. 🌌 SPATIAL UNDERSTANDING

---

## Rule

```text id="r3p7k9"
The UI must reflect real-world spatial relationships where possible.
```

---

## Implementation

* sky-based layout
* globe-based Earth
* orbital representation

---

## Failure if:

* UI feels abstract
* location/position unclear

---

# 11. ⚡ SPEED & RESPONSIVENESS

---

## Rule

```text id="x9m4p2"
The UI must feel immediate.
```

---

## Requirements

* fast transitions
* no lag
* responsive interaction

---

## Failure if:

* delays occur
* interaction feels sluggish

---

# 12. 🎛️ CONSISTENCY

---

## Rule

```text id="p2k9x7"
The same action must always produce the same result.
```

---

## Applies To

* buttons
* object clicks
* navigation
* filters

---

## Failure if:

* behavior changes unexpectedly
* interaction is inconsistent

---

# 13. 🎨 RESTRAINT

---

## Rule

```text id="v8q3m1"
Visual effects must serve clarity, not decoration.
```

---

## Must Avoid

* excessive animations
* unnecessary effects
* visual noise

---

## Failure if:

* UI feels flashy instead of functional

---

# 14. 🔦 ACCESSIBILITY

---

## Rule

```text id="k4p7x2"
The system must be usable in all environments.
```

---

## Includes

* 5-mode styling system
* readable text
* high contrast options
* red mode for night vision

---

## Failure if:

* any mode reduces usability
* readability is compromised

---

# 15. 🧠 PREDICTABILITY

---

## Rule

```text id="m2q8p5"
The user must always know what will happen before they act.
```

---

## Implementation

* clear affordances
* visible states
* consistent feedback

---

## Failure if:

* actions feel unpredictable
* UI surprises the user

---

# 16. 🚫 FORBIDDEN PATTERNS

---

The UI must NOT:

* behave like a dashboard grid
* show all information at once
* hide critical controls
* mix multiple focuses
* prioritize visuals over usability

---

# 17. 🧪 VALIDATION CRITERIA

---

UI design is correct ONLY IF:

---

## Clarity

* user understands immediately

---

## Usability

* user can navigate without confusion

---

## Guidance

* system helps decision-making

---

## Consistency

* interactions are predictable

---

## Focus

* user attention is controlled

---

# 18. 🚫 FAILURE CONDITIONS

---

The design is considered FAILED if:

* user is confused
* UI feels overwhelming
* hierarchy is unclear
* interaction is inconsistent
* system behaves like a data viewer

---

# 19. 🔥 FINAL STATEMENT

```text id="c3k9p7"
Design is not decoration.

Design is how the system communicates.

If the user cannot understand the system instantly,
the design has failed.
```
