# FEATURE SPEC TEMPLATE

## Purpose

Template for defining or reviewing one bounded feature slice.

Use this when a feature needs a concrete spec or evidence card.

---

## Feature

Name of the feature or feature slice.

## User-Visible Behavior

What the user can actually see or do.

## UI Entry

Where the user enters the feature behavior.

## API / Backend Entry

Primary route, endpoint, or backend entry path.

## Backend Ownership Path

Relevant service chain or ownership path.

## Data Sources

Authoritative sources, provider path, or internal source chain.

## Contract Shape

Relevant contract or schema references.

## Degraded / Fallback Behavior

What happens if ideal data or behavior is not available.

## Determinism Criteria

What input changes should change output, and how.

## Remaining Fake / Partial Behavior

Any current truth gaps.

## Proof

Commands, endpoint checks, screenshots, runtime observations, or other evidence.

## Status

One of:

* `REAL`
* `PARTIAL`
* `FAKE`
* `BLOCKED`
